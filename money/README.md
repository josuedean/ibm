# Money Dashboard (GitHub Pages + Google Apps Script + Google Sheets)

_Last updated: 2026-03-26_

This folder contains a mobile-first web app for recording cash transactions and displaying all-time total cash:

- Total Cash = Total Received - Total Spent
- KST timezone (`Asia/Seoul`)
- No browser session/local storage
- Public submission allowed (no authentication in v1)

---

## 1) Files in this folder

- `index.html` - UI layout
- `styles.css` - responsive styling
- `app.js` - form behavior and rendering
- `api.js` - Apps Script API client
- `config.js` - set your Apps Script Web App URL
- `apps-script/Code.gs` - server-side API for Google Sheets
- `apps-script/appsscript.json` - Apps Script manifest (KST + V8)

---

## 2) Google Sheet structure (detailed)

Create one spreadsheet with exactly two tabs named:

1. `Counterparties`
2. `Transactions`

### 2.1 `Counterparties` tab

Header row (Row 1) **must match exactly**:

| Column | Header |
|---|---|
| A | `counterparty_id` |
| B | `name` |
| C | `category` |
| D | `description` |
| E | `phone` |
| F | `active` |
| G | `created_at_utc` |
| H | `created_at_kst` |

Rules:
- `category`: `tutee` or `merchant`
- `active`: `TRUE` or `FALSE`
- IDs are generated by Apps Script (`UUID`)

### 2.2 `Transactions` tab

Header row (Row 1) **must match exactly**:

| Column | Header |
|---|---|
| A | `transaction_id` |
| B | `created_at_utc` |
| C | `created_at_kst` |
| D | `transaction_datetime_kst` |
| E | `type` |
| F | `person` |
| G | `counterparty_id` |
| H | `counterparty_name` |
| I | `counterparty_category` |
| J | `amount_krw` |
| K | `description` |

Rules:
- `type`: `received` or `spent`
- `person`: `Josh` or `Victoria`
- `amount_krw`: positive integer only
- `counterparty_category` is validated server-side:
  - `received` requires `tutee`
  - `spent` requires `merchant`

## 3) Apps Script deployment steps (manual upload)

1. Open your Google Sheet.
2. Extensions -> Apps Script.
3. Replace default script with contents of `apps-script/Code.gs`.
4. Set `appsscript.json` contents in the project manifest.
5. Save.
6. Deploy -> New deployment -> Type: Web app.
7. Execute as: **Me**.
8. Who has access: **Anyone**.
9. Deploy and copy the Web App URL.

---

## 4) Connect the frontend to Apps Script

Edit `money/config.js`:

```js
window.MONEY_APP_CONFIG = {
  APPS_SCRIPT_URL: "PASTE_WEB_APP_URL_HERE",
  TIMEZONE: "Asia/Seoul",
};
```

Commit/push to GitHub, then open your GitHub Pages URL for `/money/`.

## 6) Current functionality

- Home screen has two large centered actions: **Record Cash Received** and **Record Cash Spent**.
- Home screen displays:
  - all-time total cash (received - spent),
  - current month received cash,
  - current month spent cash.
- Pressing a record button switches to a focused form-only screen (other dashboard items hidden).
- After successful submit, a **Recording Transaction...** overlay appears while waiting on Apps Script, then app returns to home screen.
- Add new counterparties (tutee/merchant).
- Counterparty dropdown dynamically filtered by transaction type.
- Required field validation before submit and server-side validation before sheet write.
